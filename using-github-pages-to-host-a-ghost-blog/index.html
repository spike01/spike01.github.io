I finally gave in and started a blog.

Since everyone and their goat is using GitHub pages to host a blog, I figured that would be the way to go. But, as usual, instead of following the easy instructions and setting up Jekyll, I wanted more. I wanted useless speed. I wanted unused power. I wanted SHINY. (Actually, I was just a little bit jealous of Ben Forrest's blog). Long story short, I wanted Ghost.

But, Ghost doesn't run on GitHub pages. GH pages only serves static content, while Ghost needs to be served by Express, which means Node and everything else that entails. Of course, some fine folk on the Interweb had me covered.

I won't bother repeating their clear and detailed instructions here, but will share the extra little step I took to shave off a few keystrokes every time I'm ready to publish. Once you've got everything set up, rig up a little shell script called publish.sh:

#!/bin/bash

buster generate --domain=http://127.0.0.1:2368  
buster deploy

Don't forget to make it executable with chmod +x publish.sh, then whenever you want to deploy, just run ./publish.sh - no more having to remember that whole buster generate --domain=http://127.0.0.1:2368 command.

And that's exactly what I've done so I can share this with you!
